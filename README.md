### Branch & PR & Commit

*  Branch는 기능별로 구분합니다.
```
component/input
component/button
component/productbox
```
  
*  PR & Commit 메세지는 구분이 용이하도록 말머리와 함께 작성합니다. 
```
[feat] 제목     // 새로운 기능 추가
[fix] 제목      // 버그 수정
[design] 제목   // CSS 등 사용자 UI 디자인 변경
[refact] 제목   // 코드 리팩토링
[style] 제목    // 코드 포맷 변경, 세미콜론 누락 등 코드 수정이 없는 경우
[etc] 제목      // 기능 및 UI 변경 외의 사항
```
